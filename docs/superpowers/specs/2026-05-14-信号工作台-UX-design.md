# 信号工作台 — UI/UX 设计文档（v0.1）

**日期**：2026-05-14
**配套文档**：`2026-05-14-信号工作台-design.md`（产品需求 spec v0.3）
**设计方向**：B 反茧房 + Financial Dashboard 风格
**来源**：spec v0.3 + ui-ux-pro-max skill 推荐 + Ground News / Inoreader 等市场参考

---

## 一、设计定位

### 风格关键词
**Dark Mode (OLED) + Data-Dense + Accessible**

### 三个核心特征
1. **反茧房视觉**：每张话题卡片显示"立场分布条"，彩色 bar 一眼可见分歧度
2. **金融仪表盘感**：深色 + 红绿涨跌色 + 等宽字体，cycle trader 友好
3. **信息密度高**：紧凑布局、小卡片、最大化每屏可见话题数

### 不做（视觉层面）
- ❌ AI 风格的紫色/粉色 gradient（避免 anti-pattern）
- ❌ 默认 light mode（深色为默认；light mode 是 v2）
- ❌ Emoji 作为功能图标（用 SVG icon set，例 Lucide / Heroicons）
- ❌ 装饰性动画（动效必须有信息含义）

## 二、设计系统

### 2.1 颜色

基于 Financial Dashboard 调色板（dark theme），叠加反茧房立场色。

#### 基础色

| 角色 | Hex | CSS Variable | 用途 |
|---|---|---|---|
| Background | `#020617` | `--bg` | 页面主背景（OLED 黑） |
| Surface / Card | `#0E1223` | `--surface` | 卡片、面板背景 |
| Surface Elevated | `#1A1E2F` | `--surface-elevated` | 悬浮、hover 状态 |
| Foreground | `#F8FAFC` | `--fg` | 主文字 |
| Muted Foreground | `#94A3B8` | `--fg-muted` | 次要文字、时间戳、元数据 |
| Border | `#334155` | `--border` | 卡片边框、分隔线 |
| Border Subtle | `#1F2937` | `--border-subtle` | 内部分隔线 |

#### 功能色

| 角色 | Hex | 用途 |
|---|---|---|
| Primary | `#3B82F6` | 主要交互按钮、链接、焦点指示 |
| Success / Positive | `#22C55E` | 正向涨幅、确认操作、"看多"立场 |
| Destructive / Negative | `#EF4444` | 负向跌幅、删除、"看空"立场、错误 |
| Warning | `#F59E0B` | 警告状态、"宏观偏空"立场 |
| Info | `#06B6D4` | 提示信息、中性数据 |

#### 立场标签色（反茧房核心）

每个立场用一种**饱和度高的纯色 marker**（不是 gradient）。

| 立场类型 | Hex | 含义 |
|---|---|---|
| Bullish 看多 | `#3B82F6` | 蓝色 — 主流共识、看多、看好 |
| Bearish 看空 | `#EF4444` | 红色 — 看跌、警惕、悲观 |
| Skeptical 质疑数据 | `#A855F7` | 紫色 — 质疑数据 / 方法论（**注**：作为单色 marker 用，不做 gradient） |
| Macro-Bearish 宏观偏空 | `#F59E0B` | 橙色 — 宏观视角 / 跨领域看空 |
| Neutral / Data-Driven | `#22C55E` | 绿色 — 中性数据驱动 / 客观陈述 |
| Contrarian | `#06B6D4` | 青色 — 反主流 / 少数派 |

注：立场色由 AI 在话题聚类时分配，立场数量动态（不固定 4-6 种）。色块 + 文字标签同时显示，不依赖颜色单独表意（WCAG `color-not-only`）。

### 2.2 字体

```css
@import url('https://fonts.googleapis.com/css2?family=Fira+Code:wght@400;500;600;700&family=Fira+Sans:wght@300;400;500;600;700&display=swap');
```

| 用途 | font-family |
|---|---|
| Headings / 标题 | `"Fira Code", "PingFang SC", "Microsoft YaHei", monospace` |
| Body / 正文 | `"Fira Sans", "PingFang SC", "Microsoft YaHei", system-ui, sans-serif` |
| Data / Code / 数字 | `"Fira Code", "JetBrains Mono", monospace` |

**为什么用 Fira Code 做标题**：等宽字体强化 dashboard / 数据仪表盘感，配合 cycle trader 定位。中文 fallback 是 PingFang（macOS）和 微软雅黑（Windows）。

**字号体系**：

| Token | 像素 | 用途 |
|---|---|---|
| `text-xs` | 11px | 时间戳、元数据 |
| `text-sm` | 12px | 推文热度指标、次要文字 |
| `text-base` | 14px | 正文（中文略偏小但密集场景合适） |
| `text-md` | 15px | 推文翻译内容 |
| `text-lg` | 17px | 话题卡片标题 |
| `text-xl` | 19px | 详情页话题标题 |
| `text-2xl` | 24px | 页面级标题（很少用） |

### 2.3 间距

基于 4px 系统：

| Token | 值 | 用途 |
|---|---|---|
| `space-1` | 4px | 紧凑间隙（图标内边距） |
| `space-2` | 8px | 卡片内字段间距 |
| `space-3` | 12px | 卡片之间间距 |
| `space-4` | 16px | 卡片内边距 |
| `space-5` | 20px | 区域内边距 |
| `space-6` | 24px | 立场组之间间距 |
| `space-8` | 32px | 主分区之间间距 |

### 2.4 圆角

| Token | 值 | 用途 |
|---|---|---|
| `radius-sm` | 3px | 标签、徽章 |
| `radius-md` | 6px | 卡片、按钮 |
| `radius-lg` | 8px | 大容器、模态 |
| `radius-pill` | 9999px | 标签条、筛选 chip |

### 2.5 阴影

dark theme 阴影用得克制（在深色背景上视觉效果弱），主要靠 border + surface 分层：

| Token | 用法 |
|---|---|
| `shadow-card` | `0 1px 3px rgba(0,0,0,0.4)` — 卡片浅阴影 |
| `shadow-modal` | `0 10px 30px rgba(0,0,0,0.6)` — 模态深阴影 |
| `shadow-none` | 默认 — 内嵌结构层级靠 surface elevation 区分 |

### 2.6 动效

遵循 ui-ux-pro-max 的 Priority 7 准则：

| 类型 | 时长 | 缓动 |
|---|---|---|
| Micro-interaction（hover / focus） | 150ms | `ease-out` |
| State transition（卡片 hover / 选中） | 200ms | `cubic-bezier(0.4, 0, 0.2, 1)` |
| Page transition / 详情页打开 | 250ms | `ease-out` |
| Exit animations | 150ms | `ease-in` |

必须遵守：
- 只 animate `transform` 和 `opacity`（性能）
- 支持 `prefers-reduced-motion: reduce` — 用户启用减少动效时禁用所有装饰性动画
- 状态变化是 cause-effect 表达，不是装饰

## 三、页面体系

### 3.1 主工作台（Workbench Home）

**布局**：三栏式 — 左 180px + 中 自适应 + 右 280px

#### 左栏 · 类别导航

- **主入口分组**（顶部）
  - 📌 全部（话题总数）
  - ⭐ 我的收藏（收藏数）
- **类别分组**（工具自动发现）
  - Crypto · 数字
  - AI · 数字
  - 宏观 · 数字
  - 美股 · 数字
  - 科技 · 数字
  - 其他 · 数字
- **底部**（可选）：设置入口（齿轮图标）

视觉：
- 背景 `--surface`，与主区分开
- 当前选中项：`--primary` 高亮背景 + 加粗
- Hover：`--surface-elevated`
- 类别图标：Lucide icons（非 emoji）

#### 中栏 · 话题主区

**顶部筛选条**：
- 时间维度：今日 / 本周 / 本月（pill chip）
- 排序维度：重要度 / 时间（pill chip）
- 当前选中：`--primary` 填充

**话题卡片列表**（按选择排序，按重要度默认）：

每张卡片包含：
- 时间戳（左上，`--fg-muted`，text-xs）
- 中文标题（text-lg，`--fg`，2 行截断）
- **立场分布条**（卡片底部，h-1，多色 segments 拼接，宽度按推文数比例）
- 卡片整体可点 → 进入详情页

卡片样式：
- 背景 `--surface`
- 边框 `--border-subtle`
- Hover：边框变 `--primary`
- 选中（点击后）：边框变 `--primary`，背景轻微提亮

**列表是 virtualized**（话题多时性能 — Priority 3）

#### 右栏 · 赛道趋势

**面板标题**：`🔥 赛道趋势`（用 SVG icon 替代 emoji 也可，先用 emoji 占位）

每条赛道 entry：
- 赛道名（text-base，`--fg`）
- 元数据：`12 tweets · 主流共识`（text-xs，`--fg-muted`）
- 涨跌指标：`+142%` 用 `--success` / `--destructive`（text-sm，font-mono）

底部留一个**未来扩展提示框**：
- "v1 仅赛道趋势 · 未来可扩展"
- 视觉极弱（半透明、虚线边框）

### 3.2 话题详情页

**布局**：单页居中 800-880px 宽

#### 顶部 bar
- ← 返回（左）
- 话题标题（中，text-xl，加粗）
- 元数据：时间 · 类别 · "共 N 条推文"（text-xs，`--fg-muted`）
- *注：v0.2 顶部的⭐收藏按钮已删除，收藏改到每条推文*

#### 主体 · 按立场聚类的推文组

每个立场是一个 block：
- **立场标题**：text-lg，加粗，前缀是**立场色 marker**（4px 圆点或矩形条）
  - 例：`● 看多 · 周期信号`（蓝色圆点）
- **立场推文数**：右对齐 badge（`5 条`）
- **推文卡片列表**：

每条推文卡片：
- 顶部：`@author`（左，text-sm，`--fg`）+ 发布时间（右，text-xs，`--fg-muted`）
- 中部：翻译内容（text-md，`--fg`，line-height 1.55）
- 底部：
  - 左：热度指标 `♡ 1.2k · 👁 48k`（text-xs，`--fg-muted`，font-mono）
  - 右：`→ 原文` 链接 + ⭐ 收藏按钮

收藏按钮状态：
- 默认：空心⭐，`--fg-muted`
- 已收藏：实心⭐，`--warning`（金黄色）

立场组之间用**虚线分隔**（`--border-subtle` 1px dashed）。

### 3.3 收藏列表页

**入口**：左栏「⭐ 我的收藏」点击

**布局**：单栏列表，宽度同详情页

#### 顶部
- 页面标题："⭐ 我的收藏 ·  N 条"
- 筛选：按收藏时间 / 按发布时间 排序切换
- 筛选：按类别过滤（dropdown）

#### 收藏推文列表
每条推文卡片（同详情页推文卡片，但额外加）：
- **收藏时间**（顶部，右侧次要文字）
- **所属话题**（标题旁，可点击跳转到话题详情）

#### v2 预留区
顶部一个**未来路线提示** banner（淡色、可关闭）：
- "v2 路线：你收藏的观点会被自动追踪到事件验证 — 看哪些预测对了"

### 3.4 状态界面

#### 空状态
| 场景 | 文案 + 行动 |
|---|---|
| 无话题（首次使用） | "正在抓取最新内容..." + 进度指示 |
| 无收藏 | "你还没有收藏任何推文。打开任意话题，点击 ⭐ 即可收藏。" |
| 类别下无话题 | "{类别} 暂无近期讨论" |

#### 加载状态
- 列表用 **skeleton screen**（不是 spinner — Priority 3 `progressive-loading`）
- 单个话题打开用 spinner（小范围加载，更轻）

#### 错误状态
- 抓取失败：清晰说明 + 重试按钮
- 翻译失败：保留原文 + 标记"翻译异常"

## 四、关键交互流

### 4.1 主界面消费流
```
打开工作台 → 看类别（左）扫一眼分布
             → 看主区话题（中）按重要度
             → 看右栏赛道趋势确认市场温度
             → 点击话题卡片 → 进入详情页
```

### 4.2 收藏 + 验证流（v2 vision）
```
详情页阅读 → 看到值得记的观点
              → 点 ⭐ 收藏（推文层级）
              → 推文持久化到本地（含发布时间）
              → 进入「我的收藏」 — 你的记忆库
              → v2: 工具回头看历史观点 vs 事件 → 评分来源
```

### 4.3 切换类别流
```
左栏点击「Crypto」 → 中区主区刷新为 Crypto 话题
                   → 右栏赛道趋势刷新为 Crypto 赛道
                   → URL 更新（深度链接 deep linking）
```

### 4.4 反茧房自然发生
```
浏览任意话题卡片
  → 卡片底部立场分布条 → 一眼看到几种立场 + 占比
  → 点开 → 看到所有立场并列（不是"圈内 vs 圈外"）
  → 必然接触到与你认知不一致的视角
  → 反茧房在不打扰阅读节奏的情况下自然达成
```

## 六、Accessibility 检查（Priority 1, CRITICAL）

按 ui-ux-pro-max 的 Pre-Delivery Checklist：

- ✅ 颜色对比度 ≥ 4.5:1（前景白 `#F8FAFC` on 背景 `#020617` = 极高对比度）
- ✅ Focus rings 3-4px 可见（`--primary` 外环）
- ✅ 立场色不单独表意 — 同时显示色 marker + 文字标签
- ✅ Aria-labels for icon-only buttons（收藏 / 返回 / 类别图标）
- ✅ 键盘导航：Tab 顺序 = 视觉顺序（左 → 中 → 右）
- ✅ 支持 `prefers-reduced-motion`：减少所有装饰动效
- ✅ 标题层级有序（h1 → h2 → h3）
- ✅ Touch targets ≥ 44x44px（手机端时；桌面端正常 hover 友好）
